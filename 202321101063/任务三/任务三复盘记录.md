## 二、核心决策与思想

### 2.1 为什么选 PySide6 而不是 Tkinter？

| 考虑 | PySide6 | Tkinter |
|------|---------|---------|
| 外观 | 现代、可定制深色主题 | 系统原生风格、较老旧 |
| 功能 | 完整 Qt 生态 | 轻量但有限 |
| 学习价值 | 工业级 GUI 框架 | 入门级 |

选 PySide6 的核心思路：不止是交作业，更是接触真正在项目中使用的技术。

### 2.2 架构设计——核心与界面分离

```
calculator_gui.py
├── Calculator 类（任务二的核心，零改动复用）  ← 递归下降解析器
└── CalculatorGUI 类（PySide6 包装）          ← 只负责显示和交互
```

**思考：** 不把计算逻辑和 UI 混在一起。将来换界面框架（如 Web），只需替换 GUI 类，计算核心不动。

### 2.3 深色主题的设计思考

采用 Catppuccin Mocha 配色，原因是：
- 与 VS Code 深色主题统一，视觉舒适
- 不同按钮用不同颜色：红色清空、蓝色等于、紫色函数——降低误操作
- QSS 样式表集中管理，方便换肤

## 三、踩过的坑与解决

### 3.1 PySide6 安装慢（~168MB）

- 第一次用 PyPI 官方源下载极慢
- 切换清华镜像 `-i https://pypi.tuna.tsinghua.edu.cn/simple` 后秒装
- **教训：** 国内开发优先配镜像源

### 3.2 第一行多了个 `x`

文件开头 `x#!/usr/bin/env python3` 导致 VS Code 运行报 `NameError`。
原因：编辑时误触。删除多余字符即解决。
- **教训：** 写完代码至少跑一次语法检查

### 3.3 沙箱安装 PySide6 被拦截

WorkBuddy 沙箱不允许 `pip install` 直接写入系统目录。
改用 `pip install --user` 安装到用户目录绕过限制。
- **教训：** 受限环境下优先尝试 `--user` 模式

## 四、功能迭代：从无到有

| 版本 | 新增 | 驱动因素 |
|------|------|----------|
| v1 | 基础 GUI：输入框 + 按钮 + 结果显示 | 把一个命令行工具包装成可交互图形界面 |
| v2 | 计算记录（本次）：最多 20 条，可点击回填 | 实际使用需求——反复调整表达式很烦 |

### v2 迭代的思考

- 为什么 20 条？够用不占内存，列表不会太长
- 为什么放在输入框和按钮之间？操作路径短，看完结果马上能看到历史
- 为什么点击回填而不是直接重新计算？给用户修改的机会——更灵活
- 为什么用 `item.setData(Qt.UserRole, expression)`？分离展示文本和回填数据，干净
