## 二、数据表结构说明

### 2.1 `_export_progress`（导出进度表）

系统内部使用的辅助表，记录数据导出进度，共 2 条记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| table_name | TEXT | 表名 |
| last_key | TEXT | 最后导出的主键 ID |
| exported_at | TIMESTAMP | 导出时间 |

### 2.2 `adb_model_sales_summary`（型号销量汇总表）

核心业务表，**895,076 条记录**，按月份 + 型号 + 渠道维度汇总销量。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| product_model | TEXT | 产品型号 |
| product_category | TEXT | 产品品类（家用电/净水/家用气等） |
| product_class | TEXT | 产品细分类 |
| channel | TEXT | 渠道（线下/线上/官方商城等） |
| channel_ | TEXT | 渠道补充信息 |
| fin_year_month | TEXT | 财务账期（格式：YYYYMM） |
| qty_total | INTEGER | 销量合计 |
| creator / create_time | TEXT | 创建人 / 创建时间 |
| updater / update_time | TEXT | 修改人 / 修改时间 |
| deleted | INTEGER | 软删除标记（0=有效） |
| billto | TEXT | 开票方 |

### 2.3 `vw_product_sales_customer_3`（客户销量视图）

业务视图表，**819,551 条记录**，按账期 + 客户 + 型号维度，包含动销、库存、开票、下单等信息。

| 字段 | 类型 | 说明 |
|------|------|------|
| 账期 | TEXT | 财务账期（YYYYMM） |
| 客户 | TEXT | 客户代码 |
| 型号 | TEXT | 产品型号 |
| 品类 | TEXT | 产品品类 |
| 细分类 | TEXT | 产品细分类 |
| 渠道 | TEXT | 销售渠道 |
| 动销 | REAL | 动销数量 |
| 库存 | REAL | 库存数量 |
| 开票数量 | REAL | 已开票数量 |
| 开票金额 | REAL | 已开票金额 |
| 下单数量 | REAL | 下单数量 |
| 下单金额 | REAL | 下单金额 |
| 上市时间 | TEXT | 产品上市日期 |
| 客户目标总金额 | REAL | 该客户的目标总金额 |

## 六、总结

1. **数据价值高**：覆盖 8 年销售数据，包含销量、库存、开票、客户目标等完整业务链条
2. **线下为主**：超过 93% 的销售记录来自线下渠道
3. **品类集中**：家用电、净水、家用气三个品类占总记录数的约 68%
4. **客户集中度高**：TOP10 客户开票金额占总金额约 50%，存在明显客户集中现象
5. **数据安全性**：数据库文件较大（174MB）且含业务敏感信息，已加入 `.gitignore` 禁止提交到远端仓库
