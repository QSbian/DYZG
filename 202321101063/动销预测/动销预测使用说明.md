## 一、系统简介

动销预测系统是基于 OMS（订单管理系统）销售数据库构建的 **Python + PySide6** 桌面应用，能够对产品在不同渠道、品类、细分类、型号维度的动销数据进行多算法预测，并以表格形式直观展示结果。

### 核心功能

| 功能 | 说明 |
|------|------|
| 三大预测维度 | 渠道型号 / 渠道细分类 / 渠道大类 |
| 六种预测算法 | Naive / SMA(移动平均) / Median(中位数) / HW(指数平滑) / RF(随机森林) / Croston |
| 多条件筛选 | 按渠道、品类、细分类、型号关键词筛选 |
| 自动选优 | 对每个序列自动选择准确率最高的算法 |
| 结果导出 | 一键导出 CSV 文件 |

## 五、操作指南

### 5.1 选择预测维度

左侧导航栏切换三大维度：

| 导航项 | 说明 | 聚合粒度 |
|--------|------|----------|
| **渠道型号动销预测报表** | 最细粒度，按具体产品型号预测 | 渠道 + 品类 + 细分类 + 型号 |
| **渠道细分类动销预测报表** | 中等粒度，按产品子类预测 | 渠道 + 品类 + 细分类 |
| **渠道大类动销预测报表** | 粗粒度，按产品大类预测 | 渠道 + 品类 |

> 提示：也可以通过顶部「预测维度」下拉框切换。

### 5.2 设置筛选条件

| 条件 | 说明 | 示例 |
|------|------|------|
| 渠道 | 选择线上/线下 | `线下` |
| 品类 | 产品大类 | `净水` |
| 细分类 | 产品子类 | `小厨宝` |
| 型号 | 关键词搜索 | 输入 `EWH` 匹配含 EWH 的型号 |
| 预测月数 | 向前预测几个月 | 默认 5，范围 1~36 |

### 5.3 执行预测

点击「**搜索****按钮 → 系统自动：
1. 从数据库读取符合条件的动销数据
2. 按选定维度聚合为时间序列
3. 对每个序列运行 6 种算法
4. 选出准确率最高的算法及预测值
5. 填充到表格中展示

### 5.4 查看结果

表格结构：

| 列 | 说明 |
|----|------|
| 渠道 | 销售渠道（线上/线下） |
| 型号 | 具体产品型号（大类/细分类维度为空） |
| 细分类 | 产品子类（大类维度为空） |
| 预测算法 | 自动选出的最优算法名称 |
| 准确率 | 该算法的拟合精度百分比 |
| 历史月份列 | 最近几期的实际值 |
| 未来月份列 | 预测值（绿色=正值） |

**合计行**在表格最顶部，显示所有行的汇总。

### 5.5 重置与刷新

- **重置**：清空所有筛选条件和结果表格
- **刷新数据**：重新从数据库加载维度选项（如有外部更新）

### 5.6 导出结果

点击「**导出****按钮 → 选择保存位置和文件名 → 生成 UTF-8 编码的 CSV 文件。

## 六、预测算法说明

### 6.1 算法列表

| 算法 | 名称 | 适用场景 | 特点 |
|------|------|----------|------|
| **RF** | 随机森林 | 有足够历史数据的稳定序列 | 特征工程强，捕捉非线性模式 |
| **HW** | Holt-Winters 指数平滑 | 有趋势的时间序列 | 自适应趋势，适合增长型数据 |
| **Croston** | Croston 方法 | **间歇性需求**（大量零值） | 专门为零星销售设计 |
| **SMA** | 简单移动平均 | 短期平稳序列 | 计算快，抗噪性好 |
| **Median** | 中位数法 | 异常值较多的序列 | 抗异常值能力强 |
| **Naive** | 朴素法 | 基线对比 | 用最后一期值作为预测，简单有效 |

### 6.2 自动选优策略

系统对每个时间序列分别运行所有可用算法，然后：

```
算法准确率 = 100% × (1 - MAPE)
MAPE = 平均绝对百分比误差
→ 选出准确率最高的算法作为该序列的最优预测方案
```

这样不同序列可能使用不同的最优算法，实现**自适应策略选择**。

## 七、数据处理策略

### 7.1 不修改原始数据

- 使用 SQLite 只读连接
- 所有计算在内存中的 pandas DataFrame 完成
- 原始数据库文件不会被任何操作修改

### 7.2 数据预处理

| 处理步骤 | 方法 |
|----------|------|
| 缺失值处理 | 动销字段 NULL 或 <=0 的记录被过滤 |
| 时间对齐 | 所有账期统一格式 YYYYMM，缺省账期补零 |
| 聚合方式 | 同一维度组合内的动销量求和 |
| 全零序列跳过 | 总动销 < 1 的分组不参与预测 |

### 7.3 准确率评估

- 采用 **留出法**（Hold-out）：取最近约 25% 的历史数据作为测试集
- 指标：**MAPE**（平均绝对百分比误差），转换为准确率百分比显示
- 对于 Croston 这类特殊方法，采用专用评估逻辑

## 八、常见问题

**Q1：为什么某些序列的准确率较低？**
A：可能原因包括：① 数据点太少（<10 个）；② 序列波动剧烈；③ 存在异常值。建议增加历史数据量或换更稳健的算法（如 Median）。

**Q2：预测需要多久？**
A：取决于数据量和维度。渠道大类最快（几十个序列），渠道型号最慢（数千个序列）。一般 5~60 秒完成。

**Q3：可以自定义算法吗？**
A：可以。继承 `BasePredictor` 类并实现 `fit_predict()` 方法即可扩展新算法。

**Q4：如何提高预测准确率？**
A：① 缩小筛选范围（减少噪音）；② 增加预测月数让模型有更多训练数据；③ 对特定场景固定使用 HW 或 RF。

**Q5：导出的 CSV 可以用 Excel 打开吗？**
A：可以。CSV 为 UTF-8 with BOM 编码，Excel 直接打开中文正常显示。
